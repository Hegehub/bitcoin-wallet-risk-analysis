# 🔍 RiskAnalyzer Bot — профессиональный анализатор рисков крипто-кошельков

Telegram-бот для всестороннего анализа рисков криптовалютных адресов (BTC, ETH и др.). Помогает выявить подозрительную активность, происхождение средств и оценить безопасность взаимодействия с кошельком. Система включает многоуровневую защиту, подписки с оплатой в Bitcoin, двухфакторную аутентификацию и удобную админ-панель.

---

## ✨ Возможности

- **Анализ рисков** – оценка в процентах от 0 до 100 с детализацией по факторам.
- **Категоризация происхождения средств** – биржи, майнинг, DeFi, NFT, гемблинг, миксеры, даркнет.
- **Графовый анализ связей** – визуализация и выявление подозрительных паттернов (циклы, хабы).
- **Прогнозирование риска с помощью ML** – модель на основе исторических данных повышает точность оценки.
- **Мультиязычность** – поддержка русского и английского языков (определяется автоматически по языку пользователя).
- **Система подписок** – бесплатный, PRO (0.001 BTC/мес) и Business (0.005 BTC/мес) тарифы.
- **Оплата в Bitcoin** через WalletPay – быстрые и безопасные транзакции.
- **Двухфакторная аутентификация (2FA)** – TOTP (Google Authenticator) с резервными кодами, обязательная для админов.
- **Админ-панель в Telegram** – статистика, управление пользователями, мониторинг безопасности, экспорт данных.
- **Защита от атак** – WAF, rate limiting, блокировка прокси/VPN, автоматическое обновление чёрных списков.
- **Мониторинг через Prometheus** – метрики для Grafana (запросы, ошибки, кэш, ML и др.).
- **Интеграция с Sentry** – отслеживание ошибок в реальном времени.
- **Кэширование** – Redis + PostgreSQL с TTL для ускорения повторных анализов.
- **Масштабируемость** – асинхронный код, очереди задач (Celery), Docker, горизонтальное масштабирование.
- **CI/CD** – автоматическое тестирование и деплой через GitHub Actions.

- **GitHub Actions** – CI/CD

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/risk-analyzer-bot.git
cd risk-analyzer-bot
```

2. Настройка переменных окружения

Скопируйте .env.example в .env и отредактируйте под свои параметры:

```bash
cp .env.example .env
nano .env
```

Обязательно укажите:

· BOT_TOKEN – токен вашего Telegram бота
· ADMIN_IDS – ID администраторов (через запятую)
· WALLETPAY_API_KEY, WALLETPAY_STORE_ID – для приёма Bitcoin
· ETHERSCAN_API_KEY, INFURA_API_KEY – для доступа к блокчейнам
· ENCRYPTION_KEY – ключ для шифрования 2FA секретов (сгенерируйте случайную строку)
· При необходимости настройте USE_WEBHOOK, SENTRY_DSN и другие параметры.

3. Запуск с Docker (рекомендовано)

```bash
docker-compose up -d --build
```

Сервисы будут доступны:

· Бот: запущен внутри контейнера
· PostgreSQL: localhost:5432
· Redis: localhost:6379 (кэш) и localhost:6380 (очереди)
· Flower (мониторинг Celery): http://localhost:5555
· Prometheus: http://localhost:9090
· Grafana: http://localhost:3000 (логин/пароль по умолчанию admin/admin)

4. Запуск без Docker

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите Redis и PostgreSQL, затем выполните миграции БД:

```bash
python -c "from database.models import init_db, create_tables; import asyncio; asyncio.run(init_db()); asyncio.run(create_tables())"
```

Запустите бота:

```bash
python main.py
```

Для работы с вебхуками потребуется дополнительно настроить aiohttp сервер (см. config.USE_WEBHOOK).

5. Проверка работоспособности

Отправьте боту команду /start. Вы должны увидеть приветственное сообщение с выбором языка.

🔐 Безопасность

Двухфакторная аутентификация (2FA)

· Для администраторов 2FA обязательна.
· Для пользователей – опционально, но рекомендуется при совершении платежей.
· Резервные коды (8 штук) генерируются при настройке.
· При 5 неудачных попытках ввода кода – временная блокировка на 30 минут.

Защита от атак (WAF)

· Обнаружение SQL-инъекций, XSS, command injection, path traversal.
· Rate limiting – не более 100 запросов в минуту на пользователя.
· Блокировка прокси/VPN (через внешние API).
· Автоматическая блокировка при 5 нарушениях.

Чёрные списки

· Ежедневное обновление из внешних источников (CryptoscamDB, ChainAbuse).
· Адреса из чёрных списков сразу получают высокий риск.

🧠 Машинное обучение

Модель (RandomForest / XGBoost) обучается на исторических данных размеченных адресов. Используемые признаки:

· количество транзакций
· средняя сумма
· уникальные контракты
· доля ночной активности
· взаимодействие с миксерами
· центральность в графе

Для переобучения модели запустите отдельный скрипт:

```bash
python modules/ml/train_model.py
```

Путь к сохранённой модели задаётся в ML_MODEL_PATH (по умолчанию models/risk_model.pkl).

🌐 Мультиязычность

Язык пользователя определяется автоматически по настройкам Telegram. Поддерживаются:

· Русский (ru)
· Английский (en)

Для добавления нового языка:

1. Создайте каталог locales/xx/LC_MESSAGES/
2. Выполните pybabel extract -o locales/messages.pot .
3. pybabel init -i locales/messages.pot -d locales -l xx
4. Переведите строки в .po файле
5. pybabel compile -d locales

💰 Система подписок и платежи

Тарифы

Тариф Цена (BTC/мес) Лимит анализов/день Особенности
Бесплатный 0 3 Базовый анализ, только ETH
PRO 0.001 100 Все функции, multi-chain, мониторинг
Business 0.005 1000 API, white-label, приоритетная поддержка

Процесс оплаты

1. Пользователь выбирает тариф в меню /subscription.
2. Бот создаёт счёт через WalletPay и отправляет ссылку для оплаты.
3. После успешной оплаты подписка активируется автоматически (вебхук от WalletPay).
4. При необходимости можно использовать /backup_codes для новых резервных кодов.

· SERVER_USER
· SSH_PRIVATE_KEY
